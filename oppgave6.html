Git og GitHub

Git er en gratis og Open Source versjonskontrollsystem. Et versjonskontrollsystem er en type programvare som behandler filer, lagrer filene i et repository hvor du kan se på, beskrive endringer og hente filene. Repository er der filene blir lagret for endring, du har et repository som er lokalt og et som kan gå ut på nett. Repository på nettet kalles for GitHub.

Fordelene med å bruke Git er at du har tilgang til alle endringer som er gjort og hvem som har utført disse endringene. Du kan utføre flere endringer samtidig i repository og flere personer kan endre i repository samtidig. I tillegg, vis det skulle bli crash i form at to folk endrer samme fil samtidig, så kan dette løses med push og pull funksjonene.

Push vil legge filen ut på repository som den nyeste versjonen av den fila, pull lar deg hente endringer andre har gjort. Dette er viktig når flere folk endrer samme fil. Med å utføre pull så kan du se på de endringene de har gjort og utføre en merge. Dette vil slå sammen endringene som lar deg da utføre en pull med begge endringene lagt til den nyeste versjonen av fila.

For å kunne bruke et repository trenger du programvare som kan gjøre det, jeg bruker GitKraken og det er det som blir vist her.
